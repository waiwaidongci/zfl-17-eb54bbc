# 胶片分镜条核对台

一个纯前端本地应用，用来整理纸质放映胶片片段、拖拽调整放映顺序、统计总时长、筛选颜色偏移并生成试映前破损提醒。

直接打开 `index.html` 即可使用，也可以在本目录启动静态服务访问。

## 功能概览

- **录入片段**：填写片段编号、时长、颜色偏移、破损情况、缩略图和备注
- **片段模板**：保存常用配置为模板，录入时一键套用
- **放映顺序**：拖拽或上下按钮调整片段顺序
- **统计筛选**：按颜色偏移筛选、按关键词搜索、实时显示总时长和破损数
- **试映核对**：自动列出存在颜色偏移或破损的片段供放映前核对
- **导出清单**：将完整清单导出为文本文件
- **批量导入**：粘贴或上传 CSV 文本，预览确认后批量写入放映清单

## 批量导入使用说明

### CSV 格式

CSV 文件每行一条片段，共 **5 列**（最后 1 列「备注」可选），列顺序如下：

```
片段编号, 秒数, 颜色偏移, 破损情况, 备注
```

**可选值范围：**
- **颜色偏移**：`正常`、`偏红`、`偏青`、`偏黄`、`褪色`
- **破损情况**：`完好`、`轻微划痕`、`齿孔破损`、`接片松动`、`需跳过`

### 可复制的 CSV 示例

```csv
片段编号,秒数,颜色偏移,破损情况,备注
A-001,18,正常,完好,开场街景，节奏平稳
A-002,12,偏红,轻微划痕,人物近景，左侧有划痕
A-003,14,褪色,接片松动,接片位置靠近段尾
A-004,8,正常,完好,过渡镜头
A-005,20,偏青,齿孔破损,齿孔破损，放映前检查
A-006,10,偏黄,完好,暖色调场景
A-007,15,正常,需跳过,此片段需跳过，暂不使用
A-008,16,褪色,轻微划痕,旧片，整体褪色严重
```

### 导入流程

1. 点击工具栏的「批量导入」按钮
2. 选择以下任意一种方式输入 CSV：
   - **粘贴文本**：直接将 CSV 内容粘贴到文本框中
   - **上传文件**：点击「上传 CSV 文件」选择 `.csv` 或 `.txt` 文件
3. 点击「解析预览」按钮，系统将自动解析并显示预览表
4. 检查预览表中的状态：
   - ✓ **有效**：数据格式正确，可以导入
   - ⚠ **重复**：编号在 CSV 内重复或与当前清单冲突，将跳过
   - ✗ **错误**：数据格式有误，将跳过
5. 确认无误后点击「确认导入有效行」，有效片段将被添加到当前放映清单

### 导入注意事项

- **列数要求**：每行至少 4 列，第 5 列（备注）可选
- **秒数要求**：必须为正整数（如 `12`，不能是 `abc`、`-5` 或 `0`）
- **编号唯一性**：相同编号的片段只能导入一条，系统会自动检测并标记重复
- **可选值严格匹配**：颜色偏移和破损情况必须从可选值中选择，否则视为错误
- **第一行表头**：如果第一行是表头（如「片段编号,秒数,...」），系统会自动跳过
- **空白行**：空白行会被自动忽略
- **引号支持**：如果备注中包含逗号，可用双引号包裹（如 `A-009,10,正常,完好,"画面抖动,需留意"`）
- **导入追加**：导入的片段会追加到当前清单末尾，不会覆盖现有内容
- **确认机制**：导入前会弹出确认对话框，显示即将导入的有效行数

## 片段模板使用说明

### 创建模板

1. 在「片段模板」面板中填写：
   - **模板名称**（必填）：便于识别的名称，如「常规完好片」「破损近景片」
   - **默认时长秒数**：该类片段的默认时长
   - **颜色偏移**：正常 / 偏红 / 偏青 / 偏黄 / 褪色
   - **破损情况**：完好 / 轻微划痕 / 齿孔破损 / 接片松动 / 需跳过
   - **备注前缀**：可选，例如「【旧片】」「【留意】」
2. 点击「保存为新模板」完成创建

### 套用模板

模板提供两种套用方式：

1. **录入面板快速选择**：在「录入片段」面板顶部的下拉框选择模板，点击「套用」或直接选择即可将配置填入表单
2. **模板卡片一键套用**：在「片段模板」面板中，点击任意模板卡片上的「一键套用」按钮

套用模板时：
- 时长、颜色偏移、破损情况会直接覆盖当前表单值
- 备注前缀会追加到备注内容开头（如果已有内容且未以该前缀开头）

### 删除模板

点击模板卡片右上角的「×」按钮即可删除对应模板。

### 本地存储

所有模板数据与片段清单一同保存在浏览器本地（localStorage），刷新或关闭页面后不会丢失。

## 回归验证工程化流程

本项目提供了完整的回归验证流程，支持**命令行基础检查**和**浏览器端到端回归测试**两种方式，无需任何构建工具或 npm 依赖。

### 验证方式总览

| 验证方式 | 工具 | 覆盖范围 | 运行环境 |
|---------|------|---------|---------|
| 基础自动化检查 | `verify-basics.py` | 文件完整性、HTML结构、JS核心API、测试用例完整性、CSV格式 | 命令行（Python） |
| 端到端回归测试 | `e2e.test.html` | 7条关键用户路径 + 综合场景 | 浏览器 |
| 风险规则单元测试 | `risk-rules.test.html` | 风险评分规则引擎 | 浏览器 |

### 快速开始（一键运行）

```bash
# 方式一：Python 版（推荐，跨平台）
python3 scripts/run-tests.py

# 方式二：Shell 版（macOS/Linux）
bash scripts/run-tests.sh
```

运行后会自动：
1. 执行基础自动化检查
2. 启动静态文件服务器（默认端口 8000）
3. 显示测试页面访问地址

**常用参数：**
```bash
# 只运行基础检查，不启动服务器
python3 scripts/run-tests.py --no-server

# 指定端口
python3 scripts/run-tests.py --port 3000

# 自动打开 E2E 测试页面
python3 scripts/run-tests.py --open
```

### 方式一：命令行基础检查

无需浏览器，快速验证项目结构和核心 API 是否完整。

```bash
python3 scripts/verify-basics.py
```

**检查项（共 5 大类）：**
1. **文件完整性** — 检查 index.html、app.js、risk-rules.js、batch-import.js、styles.css 等必需文件
2. **HTML 结构** — 验证 script 标签引用、CSS 引用、关键 DOM 元素 ID
3. **JS 核心 API** — 静态分析核心类和函数是否存在（HistoryManager、SnapshotManager、风险规则 API 等）
4. **测试用例完整性** — 检查风险规则单元测试的测试框架、套件数量、localStorage 隔离机制
5. **CSV 示例格式** — 验证示例 CSV 文件的表头格式和数据有效性

### 方式二：浏览器端到端回归测试

在真实浏览器环境中运行完整回归用例，覆盖 7 条关键用户路径。

**启动静态服务器：**
```bash
# 方式一：Python 版
python3 scripts/start-server.py

# 方式二：Shell 版
bash scripts/start-server.sh

# 方式三：直接使用 Python 内置模块
python3 -m http.server 8000
```

**访问测试页面：**
- **E2E 回归测试**：http://localhost:8000/e2e.test.html （推荐，覆盖全部关键路径）
- **风险规则单元测试**：http://localhost:8000/risk-rules.test.html

打开页面后测试会自动运行，左侧显示测试报告，右侧显示应用预览。

### 关键用户路径（E2E 测试覆盖）

E2E 测试覆盖以下 7 条核心用户路径：

| # | 路径 | 测试内容 |
|---|------|---------|
| 1 | **新增片段** | 表单填写、添加到列表、数据持久化、撤销/重做 |
| 2 | **批量导入 CSV** | CSV 解析、有效性校验、重复检测、导入确认、导入后数据验证 |
| 3 | **调整顺序** | 上移/下移、边界情况、撤销/重做、数据一致性 |
| 4 | **修改风险规则** | 更新规则参数、规则验证、风险重算、默认值重置、备份恢复 |
| 5 | **创建快照** | 创建快照、快照列表管理、快照恢复、快照删除 |
| 6 | **备份恢复** | 数据备份导出、备份解析、数据恢复、完整性校验 |
| 7 | **生成试映报告** | 报告生成、风险片段筛选、统计数据准确性 |

此外还包含**综合场景测试**，模拟多条路径组合操作。

### 测试数据隔离

- E2E 测试运行前会自动备份所有 `zfl17` 前缀的 localStorage 数据
- 测试结束或页面关闭时自动恢复用户原始数据
- 风险规则单元测试使用 `__test_zfl17_` 前缀独立存储，互不干扰

## 直接使用（无需服务器）

本应用是纯前端零依赖项目，也可以**直接双击打开 `index.html`** 使用，无需启动任何服务器。

但请注意：
- 部分浏览器的 localStorage 在 `file://` 协议下可能有限制
- 运行测试页面**必须**通过 HTTP 服务器访问（iframe 加载限制）
